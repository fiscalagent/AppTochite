## 1. Как запускать тесты

### Автоматические (запускает Claude)

```bash
npm test           # одиночный прогон, результат в терминале
npm run test:watch # следит за изменениями файлов
```

**Файлы тестов:**
- [src/utils/backup.test.ts](../src/utils/backup.test.ts) — логика бэкапа/восстановления (30 тестов)
- [src/db/db.test.ts](../src/db/db.test.ts) — операции с базой данных (29 тестов)

**Итого: 59 автоматических тестов. Все должны быть зелёными.**

### Ручные

Выполнять в Chrome DevTools (Mobile → Pixel 5) или на Android-устройстве.

**Подготовка:** Открыть DevTools → Application → IndexedDB → Delete database → обновить страницу (seed накатится автоматически).

## 2. 🤖 Автоматические тесты

Эти тесты Claude запускает сам. Описание для понимания покрытия.

### Backup-утилиты (`backup.test.ts`, 30 тестов)

| Группа | Что проверяется |
|--------|-----------------|
| `isValidBackup` | Принимает корректный бэкап; отклоняет null, строку, пустой объект, неверную version, отсутствующий data, нарушение структуры; совместим с бэкапами без поля `meta` |
| `reviveDates` | ISO-строки конвертируются в Date; обычные строки, числа, null не трогаются; работает через `JSON.parse` |
| `buildCSV` | UTF-8 BOM в начале; разделитель `;`; CRLF между строками; экранирование `"` внутри значений; null/undefined → пустая строка; числа корректны |
| `buildSharpeningCSV` | Первая строка — заголовки с нужными колонками; заточка без камней — 1 строка данных; заточка с N камнями — N строк; статус `done`→«Готово», `accepted`→«Принят» |
| `exportBackup + restoreBackup` | Экспорт пустой БД даёт правильную структуру; экспорт сохраняет все данные; **полный цикл**: данные в БД → экспорт → очистка → восстановление → данные совпадают; `isSelf` сохраняется после восстановления; фото в base64 восстанавливаются; при ошибке транзакция откатывается (данные не затираются) |

### База данных (`db.test.ts`, 29 тестов)

| Группа | Что проверяется |
|--------|-----------------|
| `Клиенты` | CRUD (создание с id, isSelf, редактирование, удаление); каскадное удаление заточек; телефон и telegram как необязательные поля |
| `Заточки` | CRUD; статус `accepted` без `doneAt`; статус `done` с `doneAt`; stones как embedded JSON; фото в base64; удаление одного фото из массива; изменение статуса; фильтрация по `clientId`; фильтрация по `status` |
| `Камни` | CRUD; создание без гритности; все 7 типов; фильтрация кастомных |
| `Стали` | CRUD; HRC и рекомендуемый угол |
| `Ножи` | CRUD; полные данные; массовое удаление |
| `Meta` | Запись и обновление `seedVersion` |

## 4. 👤 Ручные тесты — бэкап и восстановление

Это **самый важный ручной раздел**. Прогонять при каждом значимом обновлении.

### 4.1 Подготовка тестовых данных

Перед тестированием создать:
- 2 клиентов (не «Я»)
- 2–3 заточки с разными статусами
- Минимум 1 заточку с фото «До» и «После»
- 1 кастомный камень в справочнике

### 4.2 Экспорт JSON

| # | Шаг | Ожидаемый результат |
|---|-----|---------------------|
| B-1 🔴 | Бэкап → «Сохранить бэкап (JSON)» | Файл `apptochite-YYYY-MM-DD.json` скачан |
| B-2 🔴 | Открыть файл в текстовом редакторе | Структура: `{ version: 1, exportedAt, data: { clients, sharpenings, stones, steels, knives, meta } }` |
| B-3 🔴 | Найти в JSON заточку с фото | Поля `photosBefore`/`photosAfter` — массивы base64-строк |
| B-4 🔴 | Найти кастомный камень | Присутствует с `isCustom: true` |

### 4.3 Экспорт CSV

| # | Шаг | Ожидаемый результат |
|---|-----|---------------------|
| B-5 🟡 | Бэкап → «Скачать CSV» | Файл `apptochite-sharpenings-YYYY-MM-DD.csv` скачан |
| B-6 🟡 | Открыть в Excel или Google Таблицах | Кириллица читается, все колонки присутствуют, разделитель `;` |

### 4.4 Восстановление — сценарий A: чистая БД

| # | Шаг | Ожидаемый результат |
|---|-----|---------------------|
| R-1 🔴 | Удалить всю БД (DevTools) → обновить | Только «Я» в списке клиентов |
| R-2 🔴 | Бэкап → «Выбрать файл…» → выбрать `.json` | Показывается превью: кол-во клиентов, заточек, камней |
| R-3 🔴 | «Восстановить» → подтвердить | Toast «Данные восстановлены», редирект на список клиентов |
| R-4 🔴 | Список клиентов | Все клиенты из бэкапа присутствуют, «Я» первый |
| R-5 🔴 | Открыть заточку с фото | Фото «До» и «После» отображаются |
| R-6 🔴 | Справочник → Камни | Кастомный камень из бэкапа присутствует |
| R-7 🔴 | Статусы заточек | Совпадают с бэкапом (accepted/done) |
| R-8 🔴 | Даты | `receivedAt` и `doneAt` корректные (не «Invalid Date») |

### 4.5 Восстановление — сценарий B: поверх существующих данных

| # | Шаг | Ожидаемый результат |
|---|-----|---------------------|
| R-9 🔴 | Создать нового клиента (не из бэкапа) | Клиент в списке |
| R-10 🔴 | Восстановить старый бэкап | Модалка предупреждает о замене данных |
| R-11 🔴 | Подтвердить | Новый клиент исчез, данные из бэкапа восстановлены |

### 4.6 Некорректный файл (защита от дурака)

| # | Шаг | Ожидаемый результат |
|---|-----|---------------------|
| R-12 🔴 | Выбрать произвольный `.json` без нужной структуры | Toast с ошибкой, текущие данные не тронуты |
