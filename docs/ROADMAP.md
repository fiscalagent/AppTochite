# AppTochite — Roadmap

> Документ отражает историю разработки, текущее состояние и планы.  
> Обновлено: 2026-04-24

---

## Общая цель продукта

Мобильное PWA-приложение для профессиональных заточников ножей. Два сценария:
- **Малый бизнес** — клиенты, статусы заказов, выручка
- **Личный журнал** — нулевой клиент «Я», фиксация своих заточек

Платформа: Android (90%). Работает офлайн, без сервера, данные хранятся локально (IndexedDB).

### Фаза 1 — MVP
**Статус: ✅ Завершено**

Всё, что вошло в первый рабочий релиз:

**Клиенты**
- C-1 Список клиентов
- C-2 Карточка клиента
- C-3 Форма добавления / редактирования клиента
- Нулевой клиент «Я» (`isSelf: true`) — создаётся при первом запуске, не удаляется

**Заточки**
- Z-1 Форма заточки (stepper: Приёмка → Заточка)
- Z-2 Детальная запись заточки (просмотр)
- Статусы: `accepted` / `done`, автоматическое проставление `doneAt`
- H-1 Лента заточек (история)

**Справочники**
- S-1 Камни, S-2 Стали, S-3 Ножи
- Предзаполненные данные: ~28 камней, ~100 сталей, ~200 ножей

**Инфраструктура**
- PWA: офлайн-режим, установка на устройство
- Нижняя навигация с быстрой кнопкой «+ Заточка»

## Известные технические долги

- `cleaner.html` — вспомогательная страница вне React-приложения, не интегрирована в роутинг
- Фото хранятся как `base64[]` — несмотря на сжатие при сохранении, при большом количестве фото IndexedDB может раздуться; потенциально нужен File System Access API
- Тесты (`db.test.ts`, `backup.test.ts`) не покрывают UI-компоненты и экраны — только логику БД и утилит
