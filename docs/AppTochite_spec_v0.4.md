# AppTochite — Функциональная спецификация v0.4

**Версия:** 0.4 · Апрель 2026  
**Платформа:** PWA · Android (90%) · интерфейс на русском языке

---

## 1. Обзор продукта

AppTochite — мобильное приложение для профессиональных заточников ножей. Решает задачу ведения журнала заточек: фиксация клиентов, ножей, камней, параметров заточки и рекомендаций.

**Целевая аудитория:** два сегмента с единым интерфейсом:

- **Заточник как малый бизнес** — клиенты, учёт заработка, статусы
- **Заточник «для себя»** — личный журнал, коллекция

**Ключевое решение:** оба сегмента используют единый интерфейс. Пользователь «для себя» представлен как **нулевой клиент («Я»)** — создаётся автоматически при первом запуске, не удаляется.

---

## 2. Навигация

Нижняя панель навигации (bottom navigation bar) — постоянная, доступна со всех экранов.

**Вкладки:**

- **Клиенты** — стартовый экран
- **История** — единая лента всех заточек
- **Справочники** — три под-вкладки: Камни / Стали / Ножи

**Плавающая кнопка действия (FAB):**

- Кнопка «+ Заточка» — Material Design FAB, закреплена поверх контента в правом нижнем углу
- Доступна с любого экрана
- При нажатии из карточки клиента (C-2): предзаполняет поле клиента в форме Z-1 и скрывает его

## 3. Реестр экранов

### 3.1 Клиенты

#### C-1 — Список клиентов

- Поиск по имени или телефону
- Нулевой клиент «Я» закреплён отдельной секцией сверху — не удаляется, имеет золотую корону на аватаре
- Строка клиента: аватар с инициалами, имя, телефон, счётчик заточек с разбивкой по статусам (принят / готов)
- Кнопка «Добавить клиента» — пунктирная строка внизу списка
- Подзаголовок аппбара: общее число клиентов и заточек

> Удаление клиента — только через карточку C-2, без свайпа.

#### C-2 — Карточка клиента

- Шапка: аватар с инициалами, имя, телефон, Телеграм
  - Тап по телефону → копирует в буфер, показывает toast «Телефон скопирован»
  - Тап по Телеграму → открывает `t.me/username` в браузере (редирект в приложение)
- Метрики: заточек всего, суммарная выручка
- Кнопки: «Редактировать», «+ Новая заточка»
- История заточек клиента: нож, сталь + угол, цена, статус-пилюля

#### C-3 — Добавить / редактировать клиента

- «Готово» в аппбаре — неактивен пока имя пустое
- Кнопка «Добавить / Сохранить» дублирует действие внизу формы
- «Отмена» — возврат без сохранения

| Поле | Описание | Обяз. |
|---|---|:---:|
| Имя | текстовое поле | ✱ |
| Телефон | маска `+7 (___) ___-__-__` | |
| Телеграм | префикс `@`, username без собаки | |

- Режим редактирования: внизу ссылка «Удалить клиента» (красная, без рамки) → открывает M-1

> Удаление клиента удаляет все его заточки.

---

### 3.2 История

#### H-1 — Лента заточек

- Аппбар: «История» + подзаголовок «N заточек · X ₽»
- Поиск по клиенту, марке ножа, стали, камням
- Фильтр чипами по статусу: **Все / Принят / Готов** *(статус «В работе» удалён)*
- Две метрики: заточек всего, выручка
- Лента сгруппирована по месяцам
- Строка: нож + дата, клиент, сталь + угол, цена, статус-пилюля
- Тап на строку → Z-2

---

### 3.3 Бэкап и данные

Отдельный экран, доступен из нижней навигации или раздела истории.

#### Экспорт — JSON бэкап

- Кнопка «Экспортировать данные» → стандартный Android share sheet (Телеграм, Google Drive, загрузки и т.д.)
- Формат файла: `apptochite_backup_ГГГГ-ММ-ДД.json`
- Содержимое: все клиенты, заточки, пользовательские справочники, фото в base64

> Файл может занять несколько десятков МБ из-за фото в base64.

#### Экспорт — CSV

- Кнопка «Экспортировать в CSV» → скачивание файла
- Формат: `apptochite_ГГГГ-ММ-ДД.csv`
- Содержимое: заточки с полями: № заточки, клиент, нож, сталь, угол, цена, статус, даты
- Совместим с Excel и Google Sheets

#### Импорт данных

- Кнопка «Импортировать данные» → нативный файловый picker Android
- Если данные уже есть → предупреждение: «Импорт полностью заменит текущие данные. Это действие нельзя отменить»
- Кнопки: «Заменить и восстановить» (красная) / «Отмена»
- Toast по завершении: «Данные восстановлены · N заточек, N клиентов»

### 3.4 Создание и просмотр заточки

#### Z-1 — Форма заточки (stepper: Приёмка → Заточка)

**Шаг 1 — Приёмка:**

| Поле | Описание | Обяз. |
|---|---|:---:|
| Клиент | выбор из списка. Скрыто если пришли из C-2 | ✱ |
| Марка ножа | автодополнение из справочника + создание нового | ✱ |
| Сталь | автодополнение из справочника + создание нового | |
| HRC | выбор из диапазона 50–70, одно значение или пусто | |
| Тип работы | мультивыбор чипами: **заточка / правка РК / ремонт** | |
| Дата приёма | по умолчанию сегодня | ✱ |

> После сохранения шага 1 предлагается фото «До» (можно пропустить).

**Шаг 2 — Заточка:**

| Поле | Описание |
|---|---|
| Угол заточки | числовое поле, градусы на сторону |
| Камни | автодополнение из справочника; теги с порядковым номером |
| Создать камень | мини-форма прямо в шаге 2: бренд, зернистость (необяз.), тип |
| Комментарий и рекомендации | textarea, 2–3 строки, растягивается |
| Цена | числовое поле, ₽ |
| Статус | **принят / готов**, по умолчанию «принят» |

> Статус «в работе» удалён из системы. Существует только два состояния: принят → готов.

#### Z-2 — Детальная запись (просмотр)

- «Изменить» — только в аппбаре справа
- Шапка: клиент (аватар + имя), марка ножа, чипы (тип работы, сталь, HRC)
- Статус-пилюля + цена + даты (принят / готов)
- Тип работы — чипами
- Заточка: угол, камни тегами с порядком
- Комментарий
- Фото: два слота «до» и «после»
  - Баннер «Добавьте фото до заточки» — появляется после сохранения шага 1
  - Баннер «Добавьте фото после заточки» — появляется при смене статуса на «готов»
  - Каждый баннер можно пропустить
  - Тап на фото → полноэкранный просмотр (PhotoLightbox)
  - Удаление фото — крестик прямо в детальной записи
- «Удалить заточку» — тихая красная ссылка внизу страницы, без рамки → M-1

> Дата готовности проставляется автоматически при статусе «готов».

---

### 3.5 Справочники

#### S-1 / S-2 / S-3 — Камни / Стали / Ножи

- Переключение сегментированным контролом (3 вкладки)
- Поиск в каждой вкладке
- Кнопки «Добавить камень / сталь / нож» — расположены **вверху** каждой вкладки
- Мультиселект: долгий тап → режим выделения
  - Кнопка «Выбрать все» для быстрого выделения всего списка
  - Групповое удаление выбранных позиций

**Камни — тип абразива (7 значений):**

| Значение | Описание |
|---|---|
| `galvanic` | Гальванический |
| `ao` | Оксид алюминия (белый, розовый, корундовый) |
| `kk` | Карбид кремния (чёрный корундовый) |
| `diamond` | Алмазный |
| `elbor` | Эльборный (кубический нитрид бора) |
| `natural` | Натуральный |
| `pritir` | Притир |

**Поля камня:** бренд, зернистость *(необязательна)*, тип абразива, описание

**Поля стали:** название, HRC, рекомендуемый угол, описание

**Поля ножа:** бренд, страна, сталь, рекомендуемый угол, тип, описание

## 4. Модальные оверлеи

#### M-1 — Подтверждение удаления

- Появляется при удалении клиента (из C-3) или заточки (из Z-2)
- Текст предупреждения о последствиях
- Кнопки: «Отмена» и «Удалить» (красная)

#### M-2 — Фото кромки

- Открывает камеру устройства или галерею
- Превью снятого фото
- Кнопки: «Переснять» и «Сохранить»

## 5. Сквозные UX-решения

### Toast-уведомления

- «Телефон скопирован» — при тапе на телефон в C-2
- Всплывает снизу, исчезает через 2–3 секунды, не требует действия

### Статусы заточки

| Статус | Цвет | По умолчанию |
|---|---|---|
| принят | синий (`#4A90D9`) | при создании |
| готов | зелёный (`#3DB87A`) | вручную → автоматически ставит дату готовности |

> Статус «в работе» (янтарный) удалён — не создавать, не отображать.

### Фото до / после

- Фото «До» — предлагается после сохранения шага 1 (Приёмка), можно пропустить
- Фото «После» — предлагается при смене статуса на «готов», можно пропустить
- До нескольких фото на заточку, съёмка камерой устройства
- Хранятся как `base64[]` в полях `photosBefore` / `photosAfter`

### Автодополнение

- Поля «Марка ножа», «Сталь» и «Камень» — подсказки при вводе из справочника
- Можно ввести новое значение — оно сохранится в справочнике

### Иконки

- Все иконки — встроенные SVG-компоненты (без внешних зависимостей)

## 6. Предзаполненные справочники

**Камни (~28 позиций):** DMT, King, Naniwa, Chosera, Suehiro, Shapton, Arkansas, Атома

**Стали (~100+ позиций):** японские, американские, европейские, российские, китайские, порошковые

**Ножи (~200+ позиций):** японские кухонные, немецкие, скандинавские, американские складные, российские, китайские

Справочники версионируются. При обновлении приложения новые позиции добавляются через seed-миграции (контролируются полем `seedVersion` в таблице `meta`).

## 7. База данных

**Движок:** Dexie.js (IndexedDB), локально на устройстве, текущая версия схемы **v3**.

**Таблицы:**

| Таблица | Описание |
|---|---|
| `clients` | Клиенты, включая нулевого «Я» (`isSelf: true`) |
| `sharpenings` | Заточки. Поле `stones` — embedded JSON, не отдельная таблица |
| `stones` | Справочник камней |
| `steels` | Справочник сталей |
| `knives` | Справочник ножей |
| `meta` | Служебная таблица ключ-значение (хранит `seedVersion`) |

**Важные ограничения:**
- `Sharpening.status` — только `'accepted' | 'done'`
- `Sharpening.doneAt` — обязателен только при `status === 'done'`
- `Stone.grit` — необязательное поле
- `sharpenings.stones` — поиск/фильтрация по камням требует fullscan всей таблицы

## 10. Бэклог (за рамками текущей версии)

**Высокий приоритет:**
- Быстрое создание заточки — «повторить как прошлый раз»
- Статистика по камням (fullscan `sharpenings` — учесть при проектировании)
- Фильтрация ленты заточек по камню / стали / ножу

**Средний приоритет:**
- Шаринг результата — карточка заточки как PDF или картинка для клиента
- Распознавание ножа по фото — Claude API
- Финансовая аналитика — выручка за период, средний чек
- Уведомление клиенту «нож готов»

**Низкий приоритет:**
- Облачная синхронизация — несколько устройств
- Мультипользовательский режим
- Мультиязычность
- Общие облачные справочники — пополняются сообществом

## Changelog

**v0.4 (апрель 2026) — актуализация по реализованным изменениям:**

- Статус «в работе» удалён — осталось только `accepted` и `done`
- Фильтр H-1 обновлён: «Все / Принят / Готов»
- Тип абразива камня расширен до 7 значений (`galvanic / ao / kk / diamond / elbor / natural / pritir`)

- Тип работы в форме приёмки: `заточка / правка РК / ремонт` (вместо состояния ножа)
- Создание нового камня прямо из формы заточки (шаг 2)
- Справочники: мультиселект, «Выбрать все», групповое удаление, кнопки «Добавить» перемещены вверх
- Бэкап и данные: вынесены в отдельный экран
- CSV-экспорт заточек добавлен (с полем «№ заточки»)
- FAB вместо кнопки в навигации, SVG-иконки вместо текста/эмодзи
- Золотая корона на аватаре нулевого клиента «Я»
- Деплой: GitHub Pages вместо Vercel
- БД: версия схемы v3, таблица `meta` для seed-миграций
- Связанные документы: добавлена ссылка на `ROADMAP.md`
